Workflow

How content moves through the system

Live relay state handles collaboration. Approved cleartext content can then be snapped back into committed markdown and seed files.

1. Publish

Keep the public layer simple

Posts, guides, and evergreen pages stay readable because the public site is still mostly static files.

2. Collaborate

Use relay state for live review

Accounts, submissions, comments, drafts, and entities can stay dynamic without standing up a traditional backend.

3. Snapshot

Commit approved content back to Git

The intended operating model is live relay state first, then reviewed snapshots into seed files and markdown for durable publishing.
